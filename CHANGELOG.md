# 更新日志 (CHANGELOG)

## v1.0.5 (最新版本)

### 🌟 核心功能提升 (Core Updates)
- **多项目状态完全隔离**
  - 【生成历史】与【资产库】现在完全绑定至独立项目。新建或切换不同项目时，各面板数据将保持纯净、互不干扰。解决了曾经出现的串台覆盖与内容混合的问题。
  
- **更人性化的自动保存配置**
  - 移除了传统的拖拉滑块，重新设计为直观的三挡开关（`关闭` / `15分钟` / `30分钟`）。
  - 除了保存画布节点外，自动保存现在已覆盖所有生成历史和个人资产库的实时入库。
  
- **全局 API 配置层统一重构**
  - 彻底清理了原本割裂独立的 Chat / Image / Video API 设置窗口，现已整合为单一的全局设置（`Global API Config`）。支持向前向后无缝的数据迁移。

### 🚀 骨干架构与极限抗压重构 (OOM & Performance Hotfix)
- **彻底根治因为 OOM 导致的隐性闪退（Crash to Desktop）**
  - 我们发现了画布如果承载过多高分辨率图片时，Chromium 原生内存会冲破 2GB 限制而发生闪退崩溃（Render-Process-Gone）。本次升级通过引入 `cache:copy-file` 新型混合底层读写策略，**全面剔除了 ReactFlow OOM 的元凶（Base64 内存黑洞）**！状态管理体积缩水近 1000 倍。
  
- **修复界面“掉帧卡顿” (Stuttering fixed)**
  - 取消了因用户拖动单节点时，引发大字节 Base64 结构深度复制时的巨慢 GC（垃圾回收）。
  - 实测：即使在同一个画布里摆满 100 张超高清带参考视频的图，也不会再产生过去极其折磨的卡顿或锁死现象。自动保存更加丝滑且全无察觉。

**请注意：如果您在更新后还需要使用过去“全局时期”共享的老资产，系统并没有删除它们，它们仍保留在旧版全局存储空间里。**
